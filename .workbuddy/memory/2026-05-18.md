# 2026-05-18 工作日志

## IMtoAgent 端口合并 Bug 修复

诊断并修复了用户之前合并 `:18900` → `:18899` 端口时引入的 5 个代码 bug：

1. **`/health` 路由被 Codex 抢占** — `anthropic-proxy.ts` 第 712 行 Codex 路径先匹配 GET `/health`，导致 Anthropic 自己的健康检查永远不执行。删除了 Codex 对 `/health` 的拦截。
2. **`loadSessionConfig()` 引用未定义的 `userId`** — 第 135 行 catch 块里 `userId` 不在作用域内，运行时 ReferenceError。改为使用 `customPath` 参数。
3. **`audioSend` 模式重复注册** — `capabilities.ts` 第 107-112 行和 125-130 行完全重复，导致音频消息双倍输出。删除第二个重复块。
4. **`sharedState.lastCallUsage` 不存在** — legacy `opencode.ts` 引用不存在的属性。用 `(sharedState as any).lastCallUsage` 防止 TS 编译报错。
5. **关机时 `activeRequests` 不递减导致 30s 超时** — `gracefulShutdown` 先等请求排空再关代理，但请求在等代理响应形成死锁。改为先关代理让请求快速失败，排空超时从 30s 降到 10s。

额外清理：删除 `opencode-adapter.ts.bak` 遗留文件。

启动+关机测试全部通过，`/health` 返回正确 JSON，关机秒退无卡顿。
